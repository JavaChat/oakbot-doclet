oakbot-doclet ${version}
by Michael Angstadt
https://github.com/JavaChat/oakbot-doclet

DESCRIPTION
Generates ZIP files that contain Javadoc information for OakBot to use for its "/javadoc" command.

GENERAL ARGUMENTS
-i ................ Interactive mode. Prompts the user for information about
                    library. An alternative to providing command-line
                    arguments (see below).
-v, --verbose ..... Give verbose output.
-h, --help ........ Displays this help message.

LIBRARY INFORMATION ARGUMENTS (ignored if the -i argument is given):
--maven ........... The Maven coordinates of the library.  If the library is
                    not on Maven Central, use the --src, --name, and --ver
                    arguments instead.  Format: groupId:artfactId:version.
--src ............. The path to a ZIP file, JAR file, or directory that
                    contains the library's source code. Only use if you are not
                    using the --maven argument.
--name ............ The name of the library. Only use if you are not using the
                    --maven argument.
--ver ............. The version of the library. Only use if you are not using
                    the --maven argument.                 
--excludePackages . (optional) A comma-separated list of packages not to
                    include in the Javadoc processing.
--javadocUrl ...... (optional) The base URL to the library's online Javadocs.
--javadocUrlPattern (optional) Specifies how to construct a Javadoc URL for a
                    particular class if the website uses a non-standard Javadoc
                    system.  For example, "{baseUrl}{full /}.html", will put
                    the URL specified by the --javadocUrl argument at the
                    beginning, followed by the fully-qualified class name,
                    followed by ".html".  In this example, the package names in
                    the fully-qualified class name will be separated by "/"
                    instead of dots.
--website ......... (optional) The URL to the library's website.
--prettyPrint ..... (optional) If specified, pretty-prints the XML output that
                    is generated by this tool.
--output .......... (optional) The path to where the generated ZIP file will be saved.
                    If not specified, the ZIP file will be saved to the current
                    directory with a name based on the --name and --ver
                    arguments.

EXAMPLES

java -jar oakbot-doclet.jar \  
--maven=org.jsoup:jsoup:1.8.3 \  
--javadocUrl=http://jsoup.org/apidocs/ \  
--website=http://jsoup.org/

java -jar oakbot-doclet.jar \  
--src=java8-src.zip \  
--excludePackages=com \  
--name=java \  
--ver=8 \  
--javadocUrl=http://docs.oracle.com/javase/8/docs/api/ \  
--website=http://www.oracle.com/java

java -jar oakbot-doclet.jar \
--src android-23-src.zip \
--name android \
--ver 23 \
--javadocUrl http://developer.android.com/reference/ \
--javadocUrlPattern "{baseUrl}{full /}.html" \
--website http://developer.android.com
